aerial.js

This is an early stage demo of aerial.js. The "layers" in the demo are generated on the fly by your GPU in webGL. All of the above layers are derived from a single WMS source - you can validate this by checking the network tab. Image processing is done by stacking multiple shader programs. The effects above, in order, are:

  • 1. Unaltered WMS
  • 2. Enhanced (RGB * 2.0) WMS image produced from 1
  • 3. Sharpening kernel produced from 2
  • 4. Removing pixels that are > 60% green produced from 1
  • 5. Edge detection kernel produced from 4

aerial.js is set up so that multiple shaders from multiple sources can be stacked on top of each other. In theory, any processing algorithm that can be run on e.g SNAP, QGIS etc etc can be written into aerial. The aim for aerial.js is to be an open-source, online resource where anyone with access to a WMS can process satellite imagery in real time, using a number of different shaders.

I am not an Earth Observation expert!!

The shaders used in this demo, and in general, are very basic. If you would like to contribute 1) information that can help me write useful shaders or 2) useful shaders, you are welcome to do so via GitHub.
